# Telemetry Event Schema (v1)

All events POST to `POST /e` as a single JSON object. The collector answers
`204 No Content` immediately. `text/plain` bodies are accepted (for
`navigator.sendBeacon`). Ingestion is idempotent on `eventId`.

## Envelope (every event)

| field | type | notes |
|---|---|---|
| `v` | int | schema version, currently `1` |
| `event` | string enum | discriminator; one of the types below |
| `ts` | int | client epoch **milliseconds** |
| `eventId` | string | client-generated UUID; **unique**, used for dedup |
| `account` | string | account id (`acc_...`); unknown accounts are silently dropped |
| `placementId` | string | `plc_...` |
| `adUnitPath` | string | GAM ad unit path |
| `pageUrl` | string | full page URL (stripped when consent-gated) |
| `sessionId` | string | per-pageview id generated by the engine |
| `engineVersion` | string | e.g. `2.5.0` |
| `consent` | object | `{ gdpr: bool, tcString?, usp?, gpp? }` |
| `props` | object | event-type-specific (below) |

## Event types & `props`

| `event` | fires when | `props` |
|---|---|---|
| `player_load` | engine boots & mounts | `referrer`, `viewport`, `placement` |
| `bid_request` | `pbjs.requestBids` called | `bidders[]`, `timeout` |
| `bid_response` | each bidder responds | `bidder`, `cpm`, `currency`, `status` (`bid`/`no-bid`/`timeout`/`error`), `latencyMs` |
| `auction_win` | winner chosen | `bidder`, **`cpmRaw`**, **`cpmBiased`**, `hbPb`, `floorApplied` |
| `impression` | IMA `STARTED` / `IMPRESSION` | `adId`, `creativeId`, `adDuration` |
| `ad_complete` | `ALL_ADS_COMPLETED` | `viewedPct`, `quartiles` |
| `ad_error` | `AD_ERROR` | `errorCode`, `phase`, `fallbackServed` |
| `no_demand` | empty auction / floor reject | `errorCode`, `phase`, `fallbackServed` |

### Why raw AND biased CPM

The engine adds a **floor bias** to the winning CPM before bucketing and
overwrites the `hb_pb` it sends to GAM (`player.js` ~587–606). So the price GAM
sees can be **higher** than the actual winning bid. `auction_win` therefore
carries both `cpmRaw` (true winning bid) and `cpmBiased` (what GAM was told).
Reporting must never conflate them, or it will overstate yield.

## Storage

Events land in the Postgres `events` table (flat, append-only, typed columns +
`props JSONB`). `auction_win` promotes `cpm_raw`, `cpm_biased`, `hb_pb`, `bidder`
into columns. See the README for the "columnar later" note.

## Consent handling

If `consent.gdpr` is `true` and no `tcString` is present, the configured
`CONSENT_MODE` applies:
- `anonymize` (default): store a minimal row (no `page_url`, no IP-derived
  country, no UA, no `tcString`).
- `drop`: discard entirely.
